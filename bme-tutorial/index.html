第一章 制图入门

B 简易地图制作

各部分基本视图如下：

我们看到，除了出生点之外，地图中与左侧的 Assets（元件库）中没有任何元件，一时无从下手。实际上，官方元件组已经准备好了，只是需要自己导入。

1 导入元件组

位于 File → Import Assets，双击 BallexStandardAssetsV2.0，点击下方 Import 导入。

稍等几秒，左侧项目窗口就会看到刚刚导入的所有元件。

现在只有一个元件组，今后可能会以拓展包或创意工坊的形式发布。

2 各分类含义

详细介绍请参见元件图鉴。

  • 扩展元件 01_Extras
    除了路面、钢轨、机关以外的其他元件。例如：道具球、换球器、补给物、装饰物等。
  • 机关 02_Machinery
    例如：倒板、T 板、栅栏、门、TNT 等。
  • 钢轨 03_IronRail
    包括各类钢轨和护栏。
  • 路面 04_StoneRoad
    包含各类石质路面，也包含一些特殊的路面，例如隐形方块、特殊材质方块等。
  • 木板 05_WoodBoard
  • 冰木板 06_Iceboard
    与普通木板相比摩擦系数更小。
  • 水池 07_Pool
  • 移动路面 08_MovingRoad
    包括所有种类的可移动路面，目前只支持钢轨、路面和木板。
  • 管道 09_Tube
  • 可形变元件 10_AdjustableRoad
    包括各类使用参数来设定的元件，分为整体形变元件 [A] 和分段形变元件 [S]。

3 视图调整

  • 缩放：鼠标滚轮
  • 平移：鼠标右键拖动
  • 旋转：Alt + 鼠标左键拖动

4 第一个元件

接下来，我们制作一个非常简单的地图，只需要用一块木板，一块路面，一条双轨。
从左侧元件库选择一块木板，拖入中间的场景中。在演示中，我们使用 05_WoodBoard → 1_Board → WB_2X8。

5 调整元件位置

菜单栏右侧的工具栏，从左到右依次是：撤销、还原；仅查看、移动元件、旋转元件、伸缩元件；全局参考系模式、轴心合并模式。

  • 移动元件：快捷键 W
    通过三个坐标轴箭头在单方向内移动元件（常用）；或者通过面符号在一个面内移动元件。
    量化操作：按住 Ctrl 再操作，可以以 1 为单位移动。
  • 旋转元件：快捷键 E
    通过三个有色圆弧在对应面内旋转元件。
    量化操作：按住 Ctrl 再操作，可以以 15° 为单位旋转。
  • 缩放元件：快捷键 R
    通过三个操作柄在单方向内缩放元件。可以拖动到负半轴，此时元件会变为原来的镜像。
    量化操作：按住 Ctrl 再操作，可以以相对当前大小 10% 为单位缩放。

请不要使用最外圈的白色大圆，你会得到诡异的旋转方式。

现在，请尝试使用移动工具把木板摆放到正确的位置。

6 完成初稿

再找到这两个元件，加入地图，并使用移动和旋转功能摆放成下面这样：

  • 04_StoneRoad → 2_FullSides → SFS_2X5
  • 03_IronRail → 1_Straight → R_DoubleRail

这样，我们就制作出第一张地图了！——虽然不太精良。但别忘了，还要加上两个非常重要必不可少的东西：

  • 终点：Create → SavePoints → Destination
    添加完成后，摆放到正确的位置。
  • 关卡信息：Create → Create Level Info
    记得在 Inspector（参数）窗口里写上地图信息！

关卡信息各项说明

  • MapName：地图名，可以输入中文或各种 Unicode 字符
  • AuthorName：作者，同上
  • Time：场景：白天，夜晚，地外，宇宙，朝霞，晚霞，深渊。
    地外为半重力（实际为 5/18 倍重力），宇宙为零重力。
    朝霞和晚霞场景仅有额外的光影效果，并没有特殊性质。
    深渊中冰球逐渐融化，其他球温度逐渐上升，纸球和木球随后会点燃。
  • Weather：天气：晴，多云，雨，雪，雷暴，雾。
    仅白天和夜晚支持设定天气，其它场景天气设定无效。
    雨和雷暴天气纸球会扣减耐久值。
    雾天气会大幅影响能见度，使用空格抬升视野时即看不到玩家球；雾夜还会大幅压暗照明光。
    黑夜（夜晚多云）和雾夜没有环境光照明；雷暴夜只能靠闪电获得短暂环境光照明。
  • InitialBallType：出生球种：木球，石球，纸球，冰球，钢球，橡胶球，光球。

最后，如果一切顺利，你的界面大致是这样的：

7 保存，测试，导出

随手保存！随手测图！双重保险，防死机防蓝屏防断电最有效的方法。

  • 保存：File → Save Scene
  • 测试：File → Test Map
  • 导出：File → Export .bms File

默认保存位置为：

  • Win: %USERNAME%\AppData\LocalLow\MushREB\Ballex MapEditor\MyBallexMap\Assets\
  • Mac: ~/Library/Application Support/com.MushREB.BallexMapEditor/MyBallexMap/Assets/

在 Project（项目）窗口的 Assets 根目录就能找到保存的地图，双击即可打开。

地图制作过程中，可以随时测试地图。如果关卡信息设置正确的话，场景天气等参数会实时生效。测图器的使用将在下一节讲解。

每次测试地图，以下文件夹会导出 .bpk 格式的备份，可用于版本回退和意外情况下的地图恢复：

  • Win: %USERNAME%\AppData\LocalLow\MushREB\Ballex MapEditor\Maps\
  • Mac: ~/Library/Application Support/com.MushREB.BallexMapEditor/Maps/

当你完成地图时，即可导出为 .bms 文件，用于分享或上传至创意工坊。

.bms 文件无法导入制图器再次编辑，可以安全地分享给他人，但也意味着自己应当妥善保存地图源文件。任何人都无法仅从 .bms 文件中完美还原所有信息，包括开发者 MushREB 自己！

第二章 制图基础

A 更多常用操作

回顾一下上面已经提到的操作和快捷键：

  • 缩放视图：鼠标滚轮
  • 平移视图：鼠标右键拖动
  • 旋转视图：Alt + 鼠标左键拖动
  • 移动元件：W，按住 Ctrl 量化
  • 旋转元件：E，按住 Ctrl 量化
  • 缩放元件：R，按住 Ctrl 量化

为了能够更加顺畅地制图，在继续讲解之前，我们需要掌握更多的常用操作和对应的快捷键。

  • 仅查看：Q。不使用元件操作工具，有效防止手贱 一般没什么用。
  • 撤销：Ctrl + Z
  • 还原：Ctrl + Y
  • 原位复制：Ctrl + D。会将选中的元件在原地复制一份，层级视图中这些元件临时标有 (Clone) 字样。注意：没有 Ctrl + X/C/V。
  • 聚焦：F，将选中的元件置于视图中心，并调整视图到合适大小，方便进一步操作。此外，后续视图操作也会以此为基点，如果发现调整视角很困难，也可以使用该功能一键解决。
  • 三视图：点击右上角坐标轴箭头，即可切换到对应视角。
  • 透视视图 / 正交视图：点击右上角坐标轴下方的 Persp / Ortho 即可切换。
  • 顶点对齐：V，按下开启，松开关闭。决定制图效率的功能，接下来就会讲。
  • 轴心合并模式：Z，按下切换。当选中多个物体进行旋转、缩放操作时，开启则作为整体变换，关闭则各元件独立变换。
  • 全局参考系模式：X，按下切换。开启时使用质心参考系（平行于全局参考系建系），关闭时使用自然参考系（以元件当前姿态建系）。

B 对齐与衔接

1 坐标对齐

在 Inspector（参数）窗口中，可以直接设定每个元件的坐标、姿态和尺寸。这是调整元件最暴力的方法，通过计算，你可以精确操控几乎任何元件，制作出细节近于完美的地图。
很显然，设定每个元件的参数，工程量极大——接下来会介绍更方便的方法。但它有着不可撼动的地位：

  • 确定关键元件的位置，尤其是全图第一个元件和出生点的位置关系
  • 确保全图制图体例一致，例如路灯与路面的距离保持固定
  • 确保几何结构的性质，例如两侧路灯的对称性
  • 用于难以使用其他手段对齐的场合，例如圆弧面、球类与其他物件的对齐

对于游离的元件（与其它元件没有严格的位置关系），一般推荐将坐标凑成整数或简单的有理数。
同时，由于浮点精度原因，实际参数与设想参数有微小差距（0.001% 以内）为正常现象。

现在，请设置木板坐标，使其高度合适，与出生点距离合适，轴线与出生点对齐。
推荐的参数为：X: 0, Y: -0.5, Z: 7.5。

2 顶点对齐

在 Ballex 中，大多数元件是有棱有角的——例如使用最频繁的各种路面。如果你仔细观察过钢轨，会发现它的截面其实是正八边形。我们把这些棱角称为顶点。例如，刚刚添加的大路面 SFS_2X5，八个角都是顶点。
在移动元件模式中按住 V，你会发现，拖动工具会自动吸附当前元件的一个顶点。此时再按住鼠标左键拖动元件，它就会自动吸附到另一个元件的顶点上。

在实际尝试中，你可能会发现，元件存在一些奇怪的顶点。实际上，这里的顶点指的是建模意义中的顶点，例如贴图和纹理的交界处也会产生顶点。
但容易直观理解的几何意义中的顶点，一定也是建模意义中的顶点。而且，在绝大多数的顶点对齐的操作中，使用的顶点都是几何意义中的顶点。

顶点对齐法广泛运用在各类元件的对齐中，简洁高效，大幅节省了无谓的元件移动和设定坐标的时间。在两个元件顶点对齐后，还可以为其他操作铺垫，或者调整元件坐标（最常见的是单方向 0.5 个单位的移动），快速实现自己的制图创意。
顶点对齐是制图中最常用的对齐方法，对地图细节与制图效率有着决定性的影响，各位制图者务必多加尝试，深入理解，熟练掌握！

3 顶点对齐 + 坐标调整

有些时候，顶点对齐只用来确定物体的两个或一个坐标，其余的坐标还需要手工调整。例如，如果你尝试过使用顶点对齐法对齐钢轨和路面，你会发现只有 Z: 13.5 是符合预期的，另外两个坐标并不美观，也不便于处理。

现在，请调整双轨和终点的坐标，使得地图更加美观。
推荐的参数为：

  • 双轨：X: -3.5, Y: -0.5, Z: 13.5。
  • 终点：X: -3.5, Y: -1, Z: 24。

4 路面与钢轨细节处理


以上两个样例，大家更喜欢哪一个呢？相信绝大多数人会选择后者。前者的问题可以列举如下：

  • 路面边纹多余、缺失或被拉伸
  • 路面衔接不当，有缺角有穿模
  • 远端 1×2 的路面有缺面（右侧面是空的）
  • 钢轨间衔接生硬，有空隙，且暴露出缺面

这是后者的部件拆解图。接下来会从左上开始，按行进路线介绍每一块的元件名和作用：

  • 路端：04_StoneRoad → 3_TwoSides → STS_Ends
    全边框路面 2_FullSides 的适用范围相当有限，绝大多数情况下，我们需要使用各个元件自行拼接路面，使得路面衔接更加美观。制作窄路面从路端元件开始：平路面使用 04_StoneRoad → 3_TwoSides → STS_Ends，凹路面使用 04_StoneRoad → 4_Concave → SC_Ends1。
  • 路面：04_StoneRoad → 3_TwoSides → STS_0_2
    平路面使用 04_StoneRoad → 3_TwoSides，凹路面使用 04_StoneRoad → 4_Concave，元件名中的两个参数分别为坡度角和投影长度（注意不是实际长度）。平路面与凹路面有专门的衔接件
    04_StoneRoad → 4_Concave → SC_Ends2。
  • 斜坡路面： 04_StoneRoad → 3_TwoSides → STS_15_4
    斜坡路面由平路面切变形成，能够与平路面完美衔接。因此请坚持使用斜坡路面，不要旋转平路面得到斜坡。
  • 转角： 04_StoneRoad → 6_CrossRoad → SCR_TwoRoads
    转角有专门的元件。不应当直接拼接两条直路面，这样会导致路面边纹多余或缺失。
  • 半格路端：04_StoneRoad → 9_Others → SO_3
    转角均多出了半个单位路面长度，使用半格路端可以补上另一半，使路面总长度为整数。
  • 圆弧钢轨：03_IronRail → 2_Circle → RC_90_R100 和 03_IronRail → 2_Circle → RC_90_R200
    元件名中的两个参数分别为圆心角和半径。钢轨与路面衔接时，一般都需要在顶点对齐后进行坐标调整。两根圆弧钢轨应当形成同心圆，它们的圆心坐标应当相同。
  • 圆弧衔接轨：03_IronRail → 2_Circle → RC_30_R50
    平钢轨与坡度钢轨之间，一般需要添加小半径的圆弧钢轨作为衔接轨道，避免穿模闪烁等现象。
  • 双轨：03_IronRail → 1_Straight → R_DoubleRail，缩放参数 Z: 0.5
    单轨和双轨可以通过缩放来调整长度，而不会影响纹理，其它元件请勿轻易调整缩放。单轨和双轨旋转即可得到坡度钢轨，这一点与路面不同。
  • 圆弧衔接轨：03_IronRail → 2_Circle → RC_30_R50
  • 螺旋上升轨：03_IronRail → 3_Helix → RH_360_H2_R100 和 03_IronRail → 3_Helix → RH_360_H2_R200
    元件名中的三个参数分别为圆心角、落差和投影半径。
  • 补面轨：03_IronRail → 1_Straight → R_DoubleRail，缩放参数 Z: 0.001
    03_IronRail → 2_Circle 和 03_IronRail → 3_Helix 中的所有钢轨，两端都是缺面的。如果这些缺面暴露，需要衔接非常短的单轨或双轨，补上暴露的缺面。同样，对于路面而言，如果有缺面暴露，需要使用对应的封盖 04_StoneRoad → 1_Construct → 1_Cap 补上缺面。

总结而言，遵守以下规则，有助于路面与钢轨细节的提升：

  • 避免对单轨和双轨以外的元件调整缩放，而应该使用不同长度的元件来拼接
  • 善用各类路面元件，保持路面衔接自然，边纹完整不多余
  • 适时使用衔接轨，使钢轨衔接平滑
  • 检查路面和钢轨是否缺面，并使用对应元件补面

现在，请尝试使用以上列出的元件，灵活使用已经教授的操作和方法，复刻一份样例图。
路面和钢轨是地图中最基础最常用的元件，细节处理将直接影响整张地图的细节，因此请务必重视！

5 量化移动

我们之前提到过，按住 Ctrl 使用移动工具时，可以以 1 为单位移动。许多物件的长宽高均为整数，量化移动即可快速对齐。而当 搭建大恒方尖碑 同一元件重复出现时，配合原位复制 Ctrl + D 会有奇效。
